# Playing Atari with Deep Reinforcement Learning / Human-level Control through Deep Reinforcement Learning (Mnih et al., 2013/2015)

## The Problem (Layperson)

Can a computer learn to play video games at human level using only the raw pixels on the screen, just as a human player would? No hand-crafted features, no game-specific knowledge, no access to internal game state. Just pixels in, actions out.

This had never been achieved. Previous reinforcement learning systems required carefully designed feature representations that extracted relevant information from raw sensory data. Creating these features required human expertise and was specific to each domain. Deep learning had revolutionized image recognition by learning features automatically, but combining deep learning with reinforcement learning had been considered unstable and unreliable.

## What Didn't Work (Alternatives)

Linear function approximation with hand-crafted features was the standard approach. Researchers designed features that captured relevant aspects of game states: object positions, distances, counts. This worked but required domain expertise and did not generalize across games.

Previous attempts to combine neural networks with Q-learning had failed. The "deadly triad" of function approximation, bootstrapping, and off-policy learning caused divergence. Theoretical results showed Q-learning could diverge with even linear function approximation. Nonlinear approximators like neural networks were considered even more dangerous.

TD-Gammon's success with neural networks and TD learning was considered an anomaly, perhaps due to the stochasticity of dice rolls in backgammon. Attempts to replicate its success in other domains had largely failed.

## The Key Insight

Two techniques stabilized deep Q-learning: experience replay and target networks.

Experience replay stores transitions $(s, a, r, s')$ in a buffer and samples randomly for training. This breaks correlations between consecutive samples (which destabilize learning) and allows each experience to be used multiple times (improving data efficiency).

Target networks use a separate, slowly-updated copy of the Q-network to compute targets. Instead of $y = r + \gamma \max_{a'} Q(s', a'; \theta)$, where $\theta$ are the current weights, use $y = r + \gamma \max_{a'} Q(s', a'; \theta^-)$, where $\theta^-$ are the weights from an earlier iteration. This prevents the moving target problem where the target changes with each weight update.

Together, these techniques transform an unstable learning process into a stable one.

## The Method

The Deep Q-Network (DQN) uses a convolutional neural network to map from raw pixels to Q-values for each action:

Input: 84x84x4 grayscale images (4 stacked frames for temporal information)
Architecture: Conv(16 8x8 stride 4) → Conv(32 4x4 stride 2) → FC(256) → Output(|A|)

Training procedure:
1. Observe state $s$, select action $a$ using $\varepsilon$-greedy (explore with probability $\varepsilon$)
2. Execute $a$, observe reward $r$ and next state $s'$
3. Store $(s, a, r, s')$ in replay buffer $\mathcal{D}$
4. Sample random minibatch from $\mathcal{D}$
5. Compute targets: $y = r + \gamma \max_{a'} Q(s', a'; \theta^-)$
6. Update weights by gradient descent on $(y - Q(s, a; \theta))^2$
7. Every $C$ steps, update target network: $\theta^- \leftarrow \theta$

The same architecture and hyperparameters were used across all games. Rewards were clipped to $[-1, 1]$ to allow a universal learning rate.

## The Result

DQN achieved human-level performance on 29 of 49 Atari games tested. It surpassed human experts on games like Breakout, Enduro, and Pong. It matched or exceeded all previous algorithms on every game tested.

On some games, DQN discovered strategies that surprised human observers. In Breakout, it learned to tunnel through the wall to bounce the ball behind the bricks, a strategy that maximizes score but requires patience and precision.

The same algorithm, with no game-specific tuning, learned to play diverse games: Pong (paddle control), Space Invaders (shooting), Breakout (brick destruction), Seaquest (underwater navigation). This generality was unprecedented.

## Worked Example

Consider learning to play Pong. The network receives 84x84x4 pixel arrays representing the last 4 frames.

Early training: Network outputs random Q-values. $\varepsilon = 1$, so actions are random. The paddle moves erratically, rarely hitting the ball. Occasionally the ball bounces back by chance. These rare positive rewards (point scored) get stored in the replay buffer.

After 1 million frames: Network begins to associate paddle-near-ball states with higher Q-values. $\varepsilon$ has decayed to 0.1. The paddle tracks the ball more often. Successful volleys are more common, creating more positive experiences in the buffer.

After 10 million frames: Network has learned a nearly optimal policy. Q-values accurately predict expected returns. The paddle moves precisely to intercept the ball. $\varepsilon = 0.1$ provides just enough exploration to handle edge cases.

The learned Q-function effectively encodes: "If the ball is moving toward my side, position the paddle in its path. The value of states where I can intercept is high; states where I cannot intercept have low value."

## Subtleties

The replay buffer introduces a fundamental tradeoff. Large buffers store diverse experiences but include outdated transitions from early, poor policies. Small buffers focus on recent experience but lose diversity. DQN used 1 million transitions, representing roughly the last 4 days of game time.

Frame stacking (4 frames) provides velocity information. A single frame cannot distinguish a ball moving left from one moving right. Multiple frames make the input approximately Markovian.

Reward clipping simplifies learning but loses information. In games where score magnitude matters (e.g., distinguishing between different point values), this could hurt performance. It enables a universal learning rate at the cost of game-specific optimality.

Target network update frequency $C$ is a tradeoff. Updating too often reintroduces instability. Updating too rarely means learning from stale targets. DQN updated every 10,000 steps.

The trained networks exhibited surprising internal representations. Visualizing hidden units showed that the network learned to detect game-relevant features: ball position, paddle position, brick locations. These features emerged without supervision, purely from the RL objective.

## Critical Debates

DQN versus policy gradient methods: DQN learns Q-values and derives policies; policy gradient methods learn policies directly. DQN is more sample-efficient but requires discrete actions. Policy gradients are more general but have higher variance. The field ultimately embraced both approaches for different settings.

Sample efficiency: DQN requires approximately 50 million frames (200+ hours of gameplay) to learn a single game. Humans learn from far less experience. This sparked interest in model-based methods, transfer learning, and curiosity-driven exploration.

Generalization: DQN learns each game from scratch. It cannot transfer knowledge between games. Methods like progressive networks and multi-task learning address this limitation.

The role of the replay buffer: Is it merely a variance-reduction technique, or does it fundamentally change what is being learned? Prioritized experience replay, which samples important transitions more frequently, suggests the buffer's contents matter substantially.

Why does DQN work? The theoretical foundations for deep Q-learning remain incomplete. Convergence is not guaranteed. Yet it works spectacularly in practice. Understanding this gap between theory and practice remains an active research area.
